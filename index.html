  • 关于 hashCode 与 equals

    在 Java 中，重写 equals() 必须要重写 hashCode()，那么为什么？简单来说，两个对象如果相等，则 hashCode 一定也是相同的。两个相等对象比较时，调用 equals() 会返回 true。但是由于哈希冲突，两个有着相同 hashCode 的对象也不一定相等。因此，覆盖equals()时也需要覆盖 hashCode()。
